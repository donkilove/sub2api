# 服务器负载页面规格

## 目标

在管理后台新增“服务器负载”页面，用于管理员实时查看当前部署实例的系统与容器运行状态，帮助快速判断 CPU、内存、硬盘、Docker、Go runtime、网络与依赖健康状况。

## 页面入口

- 路由：`/admin/server-load`
- 侧边栏名称：`服务器负载`
- 位置：管理后台侧边栏，靠近“运维监控”
- 权限：仅管理员可访问

## 刷新策略

- 页面加载后立即请求一次数据。
- 默认每 5 秒自动刷新一次。
- 提供手动刷新按钮。
- 请求失败时保留上一次数据，并展示错误提示。

## 后端接口

- 方法：`GET`
- 路径：`/api/v1/admin/server-load`
- 权限：管理员鉴权
- 数据来源：实时采集，不写入数据库。
- Docker 数据：优先读取 Docker Engine socket；如果不可用，接口返回 `available: false` 和不可用原因，不影响其他指标返回。

## 状态规则

- `ok`：所有已采集指标低于 warning 阈值，依赖健康。
- `warning`：任一指标达到 warning 阈值，或 DB/Redis 任一依赖不可用。
- `critical`：任一指标达到 critical 阈值。
- `unknown`：核心采集失败且无法判断状态。

默认阈值：

- CPU：`> 80%` 为 warning，`> 90%` 为 critical
- 内存：`> 80%` 为 warning，`> 90%` 为 critical
- 硬盘：`> 85%` 为 warning，`> 95%` 为 critical
- Goroutines：`> 8000` 为 warning，`> 15000` 为 critical

## 页面展示

- 顶部概览：整体状态、采集时间、进程运行时长、手动刷新按钮、自动刷新提示。
- CPU 卡片：CPU 使用率、核心数、load average、容器 CPU 使用率。
- 内存卡片：内存使用率、已用/总量、可用内存、swap。
- 硬盘卡片：根目录与数据目录用量、inode、读写速率。
- Docker 卡片：Docker 可用性、当前容器、镜像、健康状态、容器数量、容器 CPU/内存/网络/块设备 IO。
- Go runtime 卡片：goroutines、heap、GC 次数、最后 GC 时间、进程运行时长。
- 网络卡片：主网卡、收发速率、累计收发、TCP 连接状态计数。
- 依赖健康卡片：后端、DB、Redis。

## 降级与安全

- Docker socket 不可用时，页面仍展示其他指标。
- 不展示监听端口详情、连接远端地址、数据库连接串、Redis 地址、Token 或内部敏感配置。
- 采集超时或部分失败时，接口返回可用指标和局部错误说明，前端以“不可用/未知”展示。
