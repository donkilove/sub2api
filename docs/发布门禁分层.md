# 发布门禁分层

为了减少发布前的重复等待，门禁分为 `quick`、`full`、`release` 三层。默认入口是：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode quick
```

## 分层口径

| 模式 | 使用时机 | 默认检查 |
| --- | --- | --- |
| `quick` | 日常修复和小步验证 | 根据 Git 变更判断前后端；前端跑 `typecheck` + 关键 Vitest；后端跑 `go test ./...` |
| `full` | 提交或推送前 | 前端 `lint:check`、`typecheck`、全量 Vitest；后端 `go test ./...`、`golangci-lint` |
| `release` | 发布镜像前 | 默认先跑 `full`，再跑前端生产构建和 Docker 构建 |

`quick` 会参考当前未提交变更、已暂存变更；如果工作区没有变更，则参考最近一次提交的变更范围。

## 常用命令

日常快速门禁：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode quick
```

指定前端测试文件：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode quick -FrontendTest src/components/account/__tests__/AccountUsageCell.spec.ts
```

强制跑前后端 quick：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode quick -Force
```

推送前完整门禁：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode full
```

发布门禁并生成默认镜像标签：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode release
```

如果同一 commit 刚刚通过 `full`，发布前可以跳过重复的 lint/test，只保留生产构建和 Docker 构建：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode release -SkipFullChecks
```

指定镜像标签：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\gate.ps1 -Mode release -SkipFullChecks -ImageTag donki/sub2api:manual-20260620-fa5e2f3f129a
```

## 发布纪律

- 修代码阶段优先跑 `quick`，避免一开始就触发完整 Docker 构建。
- 准备推送前跑 `full`。
- 部署前如果 commit 没变，并且 `full` 已通过，可以用 `release -SkipFullChecks` 只做发布构建。
- Docker 构建默认带入 `COMMIT` 和 UTC `DATE`，便于上线后用版本信息核对镜像来源。
- `release` 只负责本地构建，不会连接服务器，也不会重启线上容器。
